Highlight #1 Version Control

   Version Control is a system that manages and records changes made to file or set of files over time, so that when needed, you or a fellow team member may recall specific versions, previously established. This is useful for comparing files from different states of the same project and gives user leeway to go back and forth whenever needed without the risk of losing all the data.

   This topic actually interests me because I am a hobbyist when it comes to graphic design and the ability to go back to a previously established creation can help my workflow, indefinitely.

   This topic is relevant to the course because we will be utilizing a Version Control System (GitHub) to maintain our files and so forth.

Highlight #2 Centralized Version Control System

   Centralized Version Control System, or CVCS, is a type of version control system that relies on a single server to host all the versioned files. This type of VCS allows multiple clients to remotely access and modify the files at hand. Very useful for projects and other team established work.

   I have no real interest or use of a centralized version control system at the moment. A majority of my work solely relies upon my individual input and not the altercations of others.

   As for its relevance to the course, since we are covering version control systems, it is essential to understand the different forms in which they come in.

Highlight #3 Distributed Version Control Systems

   Distributed Version Control System, or DVCS, is a type of version control system that not only relies on a server-side repository but also on a client-side clone. The client fully mirrors the repository allowing more leeway in case of a failure on the server side. This type of system also provides for several different kinds of workflows such as hierarchal models, due to its essentially multiple interconnected repositories.

   As mentioned for the CVCS, I have no main interest in this type of system, but it is essential to know how it works and when it can be used.

Highlight #4 Git

   Git is a specific type of version control system for tracking changes in computer files and coordinating work on files across multiple people. Mainly used by software developers to manage source code. It can, however, be used to maintain any sort of file.

   As mentioned already in previously highlighted subjects, I have no interest in Git or its applications due to my lack of need to achieve a multi-sourced project.

   Although due to this course and its usefulness of vision control systems, it is vital for me to know and understand this subject matter fully and how one day I can fully utilize it.
